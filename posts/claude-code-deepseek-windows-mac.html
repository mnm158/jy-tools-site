先说明：桌面端和 Claude Code 不是一回事

Anthropic 官方现在提供图形化桌面端，也提供 Claude Code 命令行工具。桌面端适合聊天和打开 Claude Code 体验；真正把请求切到 DeepSeek，主要是通过 Claude Code 命令行的环境变量完成。

推荐做法： 新手先安装桌面端，确认账号和基础环境没问题；然后安装 Claude Code 命令行，再配置 DeepSeek。

macOS 部署

  1. 打开 Anthropic 官方页面下载 macOS 桌面端，安装后登录账号。
  2. 打开“终端”，安装 Claude Code 命令行：
curl -fsSL https://claude.ai/install.sh | bash

如果你平时使用 Homebrew，也可以用下面的方式安装稳定版：

brew install --cask claude-code

安装完成后，检查版本：

claude --version

能看到版本号，就说明 Claude Code 已经装好。以后在项目文件夹里运行 claude 就能启动。

Windows 部署

  1. 先安装 Windows 桌面端并登录。
  2. 建议安装 Git for Windows。官方说明里提到，原生 Windows 下有 Git Bash 会更适合 Claude Code 使用。
  3. 打开 PowerShell，执行安装命令：
irm https://claude.ai/install.ps1 | iex

如果你习惯 WinGet，也可以使用：

winget install Anthropic.ClaudeCode

安装完成后检查版本：

claude --version

如果你做的是 Linux 工具链项目，也可以选择 WSL 2。普通 Windows 软件、脚本、插件项目，用原生 Windows + Git for Windows 通常更直观。

接入 DeepSeek 模型

先到 DeepSeek Platform 创建 API Key。下面的命令里不要把 <your DeepSeek API Key> 原样复制，必须替换成你自己的 Key。

macOS 临时生效

export ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic
export ANTHROPIC_AUTH_TOKEN=<your DeepSeek API Key>
export ANTHROPIC_MODEL=deepseek-v4-pro[1m]
export ANTHROPIC_DEFAULT_OPUS_MODEL=deepseek-v4-pro[1m]
export ANTHROPIC_DEFAULT_SONNET_MODEL=deepseek-v4-pro[1m]
export ANTHROPIC_DEFAULT_HAIKU_MODEL=deepseek-v4-flash
export CLAUDE_CODE_SUBAGENT_MODEL=deepseek-v4-flash
export CLAUDE_CODE_EFFORT_LEVEL=max

Windows PowerShell 临时生效

$env:ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"
$env:ANTHROPIC_AUTH_TOKEN="<your DeepSeek API Key>"
$env:ANTHROPIC_MODEL="deepseek-v4-pro[1m]"
$env:ANTHROPIC_DEFAULT_OPUS_MODEL="deepseek-v4-pro[1m]"
$env:ANTHROPIC_DEFAULT_SONNET_MODEL="deepseek-v4-pro[1m]"
$env:ANTHROPIC_DEFAULT_HAIKU_MODEL="deepseek-v4-flash"
$env:CLAUDE_CODE_SUBAGENT_MODEL="deepseek-v4-flash"
$env:CLAUDE_CODE_EFFORT_LEVEL="max"
注意： 临时环境变量只在当前终端窗口生效。确认能跑通以后，再考虑写入 ~/.zshrc、PowerShell Profile 或系统环境变量。

DeepSeek 官方说明里，deepseek-chat 和 deepseek-reasoner 将在 2026-07-24 废弃；新配置优先使用 deepseek-v4-pro 和 deepseek-v4-flash。

验证是否成功

进入一个测试项目文件夹，然后启动 Claude Code：

cd /path/to/my-project
claude

可以输入这句话测试：

请读取当前项目结构，用中文总结这个项目是做什么的，不要修改文件。

如果它能正常回答，并且 DeepSeek 控制台里能看到 API 用量，就说明 Claude Code 已经走 DeepSeek 后端了。

常见问题

1. Windows 提示命令不可用

先确认你用的是 PowerShell 还是 CMD。PowerShell 提示符一般以 PS C:\ 开头，CMD 则没有 PS。两者安装命令不一样。

2. 输入 claude 后提示找不到命令

关闭终端重新打开，再执行 claude --version。如果还不行，检查安装过程有没有报错，或者 PATH 是否已经刷新。

3. API Key 不要放进项目文件

不要把真实 Key 写进文章、截图、视频或 GitHub 仓库。建议用环境变量保存，并在项目的 .claude/settings.json 里禁止读取 .env 等敏感文件。 

{
  "permissions": {
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  }
}

4. 模型名报错

优先检查 DeepSeek 平台当前支持的模型名和你的账号权限。如果 deepseek-v4-pro[1m] 不可用，可以先改成 deepseek-v4-pro 或 deepseek-v4-flash 验证。
