Claude Code / DeepSeek

Claude Code 桌面端如何用安装包部署，并接入 DeepSeek 模型

这篇文章按小白也能跟着做的方式，避开命令行安装，直接用官方安装包完成 Windows / macOS 部署，再用桌面端的环境配置接入 DeepSeek。

先说明：这篇按“安装包方式”来写

Anthropic 官方现在把 Chat、Cowork 和 Claude Code 放进同一个 Claude Desktop 桌面应用里。安装桌面端后，打开顶部的 Code 标签页，就可以用图形界面操作 Claude Code。

推荐做法： 新手不要复制网上来路不明的安装命令，直接从 claude.com/download 下载官方安装包。

先下载官方安装包

  1. 打开 Claude 官方下载页：https://claude.com/download。
  2. macOS 用户点击 Download for macOS。
  3. Windows 用户点击 Download for Windows；如果是 Windows ARM64 设备，选择 ARM64 版本。
  4. 下载完成后，只打开官方站点下载的安装文件，不要打开别人发来的安装包。

Claude Desktop 桌面端支持 macOS 和 Windows。Linux 暂时没有桌面端，需要使用命令行版本。

安装前准备：Windows 先确认 Git 和虚拟机平台

Windows 用户最容易漏掉这一步。Claude Code 不是普通聊天软件，它会读取项目、运行工具、查看文件变化，所以建议先把 Git 和 Windows 虚拟化能力准备好。

1. 本地 Windows 路线：先装 Git for Windows

  1. 打开 Git for Windows 下载页，下载安装包。
  2. 安装时新手保持默认选项即可，重点是保留 Git Bash。
  3. 安装完成后，完全退出 Claude Desktop，再重新打开。
为什么要装 Git： Claude 官方文档建议原生 Windows 使用 Git for Windows。装好后，Claude Code 可以使用 Git Bash 来执行代码项目里的常见操作；很多项目检查改动、回退修改、查看历史也都依赖 Git。没有 Git 时，Claude Code 可能退回 PowerShell，或者在某些 Code 会话里提示 Git required。

2. WSL 路线：需要打开虚拟机平台

如果你要用 WSL 2，也就是在 Windows 里跑 Linux 环境，需要开启 Windows 的虚拟化相关功能。普通插件、网页、小工具项目可以先走上面的 Git for Windows 路线；只有 Linux 工具链、Docker、WSL 项目才优先走 WSL。

  1. 打开开始菜单，搜索 启用或关闭 Windows 功能。
  2. 勾选 Virtual Machine Platform。
  3. 如果你要使用 WSL，也勾选 Windows Subsystem for Linux。
  4. 点击确定，按提示重启电脑。
  5. 如果仍然报虚拟化错误，需要进电脑 BIOS/UEFI 里开启 Intel VT-x 或 AMD-V。不会操作时不要乱改 BIOS，先找熟悉电脑的人帮忙。
新手建议： 你只是想用 Claude Desktop 的 Code 标签页改 Windows 本地项目时，先安装 Git for Windows 就够了；不要一上来折腾 WSL。只有看到 WSL、Docker、Linux 工具链相关报错，再处理 Virtual Machine Platform。

macOS 安装

  1. 双击下载到的 Claude 安装文件。
  2. 把 Claude 拖到“应用程序”文件夹，或按安装器提示完成安装。
  3. 从“应用程序”里启动 Claude，并登录你的 Anthropic 账号。
  4. 在顶部切换到 Code 标签页。
  5. 第一次使用时，选择 Local，再选择你的项目文件夹。

大多数 Mac 自带 Git。后面如果 Code 标签页提示找不到 Git，再安装 Xcode Command Line Tools 或 Git 客户端即可。

Windows 安装

  1. 双击下载到的 Claude Windows 安装包。
  2. 按安装器提示完成安装；如果弹出 UAC 权限确认，确认发布者和来源后再继续。
  3. 确认已经安装 Git for Windows。如果刚装完 Git，需要先完全退出 Claude Desktop。
  4. 从开始菜单启动 Claude，并登录你的 Anthropic 账号。
  5. 重启 Claude Desktop，打开顶部 Code 标签页。
  6. 第一次使用时，选择 Local，再选择你的项目文件夹。

普通 Windows 软件、脚本、插件项目，用 Claude Desktop + Git for Windows 最直观。只有 Linux 工具链项目才优先考虑 WSL。

和网上教程的差异： 有些 Windows 教程会让你先安装 Node.js 18+。按 Claude 官方桌面端快速开始文档，Code 标签页已经内置 Claude Code，不需要为了桌面端单独安装 Node.js 或 CLI。只有你要在终端里使用 claude 命令时，才需要另外安装 CLI。

接入 DeepSeek 模型

先到 DeepSeek Platform 创建 API Key。桌面端不需要用终端 export 变量，可以直接在 Code 标签页的 Local 环境编辑器里填写。

在桌面端填写环境变量

  1. 打开 Claude Desktop 的 Code 标签页。
  2. 在输入框附近找到 Environment 下拉菜单。
  3. 鼠标放到 Local 上，点击齿轮图标，打开 Local environment editor。
  4. 逐条新增下面这些变量。左边填变量名，右边填变量值。
  5. 保存后新建一个 Local 会话，或重启当前会话让配置生效。
ANTHROPIC_BASE_URL = https://api.deepseek.com/anthropic
ANTHROPIC_AUTH_TOKEN = <your DeepSeek API Key>
ANTHROPIC_MODEL = deepseek-v4-pro[1m]
ANTHROPIC_DEFAULT_OPUS_MODEL = deepseek-v4-pro[1m]
ANTHROPIC_DEFAULT_SONNET_MODEL = deepseek-v4-pro[1m]
ANTHROPIC_DEFAULT_HAIKU_MODEL = deepseek-v4-flash
CLAUDE_CODE_SUBAGENT_MODEL = deepseek-v4-flash
CLAUDE_CODE_EFFORT_LEVEL = max
注意： <your DeepSeek API Key> 必须换成你自己的 Key。录屏、截图、发文章时都要遮住真实 Key。

Windows 可选：CC Desktop Switch 图形界面方案

如果你更想要“下载安装包、点按钮配置”的体验，可以了解 CC Desktop Switch。它是第三方开源工具，用来把 DeepSeek、Kimi、智谱等 Anthropic 兼容接口写入 Claude Desktop 的网关配置。

  1. 先完全退出 Claude Desktop。Windows 需要在系统托盘右键 Claude 图标，选择退出。
  2. 从 CC Desktop Switch Releases 下载 Windows Setup 安装包。
  3. 安装后打开工具，选择或新增 DeepSeek 提供商。
  4. 接口地址填 https://api.deepseek.com/anthropic，API Key 填你自己的 DeepSeek Key。
  5. 按需要启用 1M 上下文、Max 思考或模型映射选项。
  6. 点击应用到 Claude Desktop，按提示重启 Claude Desktop。
  7. 如果登录页出现 Continue with Gateway，选择它进入。
第三方工具提醒： CC Desktop Switch 不是 Anthropic 官方产品。它的 GitHub 说明也提到 Windows 安装包可能没有 Authenticode 签名，安装时可能出现未知发布者提示。只从官方 GitHub Releases 下载，并确认你能接受这个风险后再使用。

Windows 可选：第三方推理配置入口

有些 Windows 教程提到，部分 Claude Code Desktop 版本可以在开发者模式里直接配置第三方推理服务。如果你的桌面端确实出现了这个入口，可以这样填；如果没有看到这个菜单，不要强行寻找，继续使用上面的 Local environment editor 即可。

  1. 在 Claude Desktop 菜单里进入 Help → Troubleshooting，查看是否有 Enable Developer Mode。
  2. 重启后如果出现 Developer → Configure Third-Party Inference，打开配置窗口。
  3. Gateway base URL 填 https://api.deepseek.com/anthropic。
  4. Gateway API key 填你的 DeepSeek API Key。
  5. Gateway auth scheme 选择 bearer。
  6. Model list 添加 deepseek-v4-pro[1m]，保存后重启 Claude Desktop。

这个入口不同版本、账号和企业配置下可能不一样，所以本文仍然把 Local environment editor 作为更稳妥的主流程。

DeepSeek 官方说明里，deepseek-chat 和 deepseek-reasoner 将在 2026-07-24 废弃；新配置优先使用 deepseek-v4-pro 和 deepseek-v4-flash。

日常使用建议

先建一个测试工作目录

第一次使用 Code 模式时，不要直接授权桌面、下载目录或整个硬盘。建议新建一个测试文件夹，把要试的项目复制进去，再让 Claude 读取和修改这个文件夹。

权限模式怎么选

模式 适合场景 新手建议
Ask permissions 每次改文件或运行命令前都询问 第一次使用优先选它
Plan mode 先分析、列计划，不直接动文件 复杂任务先用它审方案
Accept edits 自动接受常规文件编辑 熟悉项目后再用
Bypass permissions 尽量减少确认步骤，自动执行更多操作 只在可重建的测试目录里用

模型和思考等级

日常写代码、查问题可以先用 deepseek-v4-pro 加 High 思考。简单问答、改小文本可以用 deepseek-v4-flash 控制成本。长链路调试、复杂项目迁移，再考虑 1M 上下文和 Max 思考。

API Key 和余额

DeepSeek API Key 创建后通常只显示一次，复制后要妥善保存。调用 API 会消耗余额，如果 Claude 能打开但一直报鉴权或额度错误，优先检查 Key 是否填错、账户是否有余额。

验证是否成功

在 Code 标签页里选择一个测试项目文件夹，建议先用“Ask permissions”权限模式。然后输入这句话测试：

请读取当前项目结构，用中文总结这个项目是做什么的，不要修改文件。

如果它能正常回答，并且 DeepSeek 控制台里能看到 API 用量，就说明 Claude Code 已经走 DeepSeek 后端了。

常见问题

1. Code 标签页打不开或提示需要升级

先确认 Claude Desktop 是最新版本，并确认账号套餐支持 Claude Code。官方文档说明 Claude Code 桌面端需要 Pro、Max、Team 或 Enterprise 订阅。

2. Windows 提示 Git required

安装 Git for Windows，安装完成后完全退出 Claude Desktop，再重新打开。Claude 官方文档建议原生 Windows 使用 Git for Windows；有 Git Bash 时，Claude Code 会优先用它处理项目里的命令和 Git 操作。

3. Virtual Machine Platform not available

这是 Windows 虚拟机平台功能没有开启，常见于 WSL 2 或依赖 Linux 环境的项目。打开“启用或关闭 Windows 功能”，勾选 Virtual Machine Platform，需要 WSL 时同时勾选 Windows Subsystem for Linux，然后重启电脑。如果仍然报错，再检查 BIOS/UEFI 里的虚拟化是否开启。

4. CC Desktop Switch 应用后没变化

先确认 Claude Desktop 已经完全退出，而不是只关闭窗口。新版 CC Desktop Switch 默认通过本机网关工作，所以使用第三方提供商时也要让它留在后台运行。

5. API Key 不要放进项目文件

不要把真实 Key 写进文章、截图、视频或 GitHub 仓库。建议用环境变量保存，并在项目的 .claude/settings.json 里禁止读取 .env 等敏感文件。 

{
  "permissions": {
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  }
}

6. 模型名报错

优先检查 DeepSeek 平台当前支持的模型名和你的账号权限。如果 deepseek-v4-pro[1m] 不可用，可以先改成 deepseek-v4-pro 或 deepseek-v4-flash 验证。

7. 仍然想用命令行怎么办

命令行安装仍然是官方支持的备用方案，但这篇文章不把它作为主流程。你只想用安装包的话，按上面的 Claude Desktop 流程即可。

配套视频内容脚本

  1. 开场：说明本期只用安装包，不用命令行安装 Claude Code。
  2. 展示下载页：打开 claude.com/download，分别指出 macOS、Windows、Windows ARM64。
  3. 展示 macOS 安装：下载、打开安装包、启动 Claude、进入 Code 标签页。
  4. 展示 Windows 安装前准备：先安装 Git for Windows，说明 Git Bash 和 Git required 的关系。
  5. 补充 WSL 场景：如果走 WSL 2，需要在 Windows 功能里开启 Virtual Machine Platform，并重启电脑。
  6. 展示 Windows 安装：下载、运行安装包、确认 Git 已安装、重启 Claude、进入 Code 标签页。
  7. 展示 DeepSeek：进入平台创建 API Key，但录屏时遮住 Key。
  8. 展示桌面端 Local 环境编辑器：逐条填写 DeepSeek 环境变量。
  9. 补充 Windows 可选路线：展示 CC Desktop Switch 下载、添加 DeepSeek、应用到 Claude Desktop。
  10. 讲清楚权限模式：新手先用 Ask permissions 或 Plan mode，不要一开始就开 Bypass permissions。
  11. 进入测试项目，让 Claude 只读取项目并总结，不修改文件。
  12. 结尾：提醒不要泄露 API Key，后续可把配置写入启动脚本或系统环境变量。
