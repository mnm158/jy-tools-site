CC Switch / DeepSeek / Codex

OpenAI Codex 一站式接入 DeepSeek 等主流大模型

这篇文章按 Windows 新手也能跟着做的方式，从 OpenAI Codex 下载、DeepSeek API 准备到 CC Switch v3.16.1 配置，讲清楚如何把 Codex 的模型请求切到 DeepSeek、通义千问等主流大模型，同时尽量保留 Codex 手机远程操作、官方插件和 tool_search 能力。

为什么要用 CC Switch 接 DeepSeek

OpenAI Codex 原生面向 OpenAI 的 Responses API 和 GPT 系列模型。DeepSeek、通义千问、Kimi、GLM、MiniMax 等上游通常提供 OpenAI Chat Completions 兼容接口。CC Switch v3.16.x 的关键价值，就是用本地代理把 Codex 的 Responses 请求转换成 Chat Completions，再把上游返回转换回 Codex 能理解的 Responses 形态。

需求 不用 CC Switch 使用 CC Switch
切换模型供应商 手动改配置，容易写错路径或 token 图形界面新增供应商，支持热切换和备份
接入 DeepSeek Codex 不一定能直接吃 Chat Completions 上游 本地路由做 Chat ↔ Responses 转换
保留官方能力 第三方 token 可能覆盖官方登录态 v3.16.1 可手动开启 Codex 应用增强，保留官方 OAuth

  1. 安装或更新 OpenAI Codex，并确认可以正常登录官方账号。
  2. 准备 DeepSeek API Key。只在 CC Switch 或环境变量里填写，不要写进公开网页、截图和 GitHub 仓库。
  3. Windows 用户建议先准备 Git for Windows、Node.js LTS 和 Virtual Machine Platform。前一篇 Claude Code 教程里已经解释了这三项的作用。
  4. 从官方渠道下载 CC Switch。不要使用要求充值、收费或索取登录凭据的山寨站。
安全提醒： CC Switch 官方发布说明强调，软件免费开源，下载优先走官网、GitHub Releases 或官方 SourceForge 镜像。任何要求你付费或交出账号密码的网站都要避开。

下载安装 OpenAI Codex

  1. Windows 用户打开 Microsoft Store 的 Codex 下载页。
  2. 点击 获取 或 安装，按 Microsoft Store 提示完成安装。
  3. 安装后启动 Codex，使用 OpenAI / ChatGPT 账号登录。
  4. 首次使用时先打开一个测试项目，确认 Codex 能正常创建会话、读取项目和显示模型列表。
下载建议： Windows 端优先走 Microsoft Store。不要从陌生网盘、第三方打包站或来路不明的 npm 包下载所谓 Codex 客户端。

下载安装 CC Switch

  1. 打开 CC Switch 官方站或 GitHub Releases。
  2. Windows 选择 CC-Switch-v3.16.1-Windows.msi，macOS 选择对应 .dmg，Linux 选择 .deb、.rpm 或 AppImage。
  3. 如果国内网络访问 GitHub 不稳定，可以用本站备用安装包：下载 CC-Switch-v3.16.1-Windows.msi。
  4. 安装后启动 CC Switch，第一次打开先看它是否识别到 Codex。
  5. 如果识别不到 Codex，先确认 Codex 已经安装并能从终端或桌面端正常启动。

添加 DeepSeek 供应商

  1. 在 CC Switch 左侧选择 Codex。
  2. 点击新增供应商，优先选择内置 DeepSeek 预设；如果没有预设，就选择 OpenAI-compatible 或 Custom。
  3. Base URL 填 DeepSeek 的 OpenAI 兼容地址，通常是 https://api.deepseek.com/v1。
  4. API Key 填你的 DeepSeek Key。
  5. 模型映射里把 Codex 侧模型映射到 DeepSeek 模型，例如把编码任务模型映射到 deepseek-chat 或 deepseek-reasoner。
  6. 如果页面里有 Needs Local Routing 或类似“需要本地路由”的选项，连接 DeepSeek 这类 Chat Completions 上游时建议开启。
关键点： DeepSeek 不是 OpenAI Responses API 原生供应商。CC Switch 的本地路由会在中间做格式转换，所以不要只改 base URL 就结束，还要确认本地路由和模型映射是否生效。

CC Switch v3.16.1：开启 Codex 应用增强

v3.16.1 新增了更谨慎的 Codex 增强开关，默认关闭。开启后，CC Switch 会尽量把官方 ChatGPT / Codex OAuth 登录态保留在 auth.json，第三方供应商 token 写到 provider 作用域配置中。这样做的目标是：模型流量走 DeepSeek，但官方登录相关能力继续保留。

  1. 先用官方账号登录 Codex / ChatGPT，并确认官方 Codex 能正常启动。
  2. 打开 CC Switch 设置，找到 Codex 应用增强 或类似选项。
  3. 手动开启“官方认证保留”相关开关。
  4. 回到 Codex 应用，选择 DeepSeek 供应商，并开启本地路由接管。
  5. 重启 Codex，让模型目录和 provider 显示刷新。
为什么这一步重要： v3.16.1 发布说明明确提到，开启增强后可以在使用第三方 API 的情况下继续使用 Codex 的手机远程操作、官方插件等能力；同时它修复了第三方供应商切换时官方 OAuth、模型目录、tool_search、插件 / 连接器命名空间和自定义工具恢复的问题。

验证是否成功

  1. 在 Codex 里打开一个测试项目，不要一开始就授权重要目录。
  2. 输入：请读取当前项目结构，用中文总结，不要修改文件。
  3. 观察 DeepSeek 控制台是否出现 API 调用。
  4. 在 Codex 里测试 tool_search、官方插件或连接器是否还能被发现。
  5. 打开 ChatGPT 手机 App 的 Codex 入口，检查是否还能远程查看或继续电脑上的 Codex 会话。

如果 DeepSeek 有用量记录，Codex 也能继续使用手机远程和插件能力，说明这条链路基本跑通。

常见问题

1. 开了 DeepSeek 后手机远程操作消失了

先确认你用的是 CC Switch v3.16.1 或更新版本，并且在设置里手动开启了 Codex 应用增强 / 官方认证保留。默认关闭时，旧式写入方式可能无法保留全部官方能力。

2. 模型列表为空或切换后不刷新

v3.16.1 修复了模型目录被清空的多个场景，但修改模型映射后仍建议重启 Codex，因为 Codex 启动时会读取模型目录。

3. 插件、tool_search 或自定义工具不可用

这通常是 Chat Completions 上游返回和 Codex Responses 工具事件之间的映射问题。v3.16.1 已针对 tool_search、插件 / 连接器命名空间和自定义工具做恢复修复，优先升级到这个版本再排查。

4. 这样做有没有风险？

有。CC Switch 发布说明提醒，第三方供应商 Chat 路由、OAuth 反向代理和 Claude Desktop 第三方供应商代理都需要遵守目标供应商的计费、合规和数据留存规则。生产项目、公司代码或敏感文件不要随便接到不熟悉的中转服务。
